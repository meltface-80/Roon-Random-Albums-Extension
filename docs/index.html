MusicD Remote

MusicD Remote a music discovery companion for Roon

Rediscover your library through random album browsing with rich metadata, beautiful wall displays and seamless playback — with Roon Server at the heart.

v1.6.35 Docker install MIT license Releases ↗

Features

Designed for large libraries, and friendly on phones, tablets, desktops and TVs.

🎵Album discovery

Random albums, Album of the day, Label of the week, unheard-album picks and recently-unplayed recommendations. Over time it learns what you've heard and surfaces the forgotten corners of your library.

📚Rich browsing

Browse by album, artist, genre, record label, decade or tag — and jump between related artists, albums and labels from anywhere.

🔍Powerful search

Instant whole-library search, optionally extended to Qobuz and TIDAL. Browse the streaming catalogues and add favourites straight into your Roon library.

▶️Playback control

Play or queue albums and individual tracks, multi-select, switch zones, and let Random Album Radio keep the music flowing when the queue runs out.

📺Wall display

Turn a TV or tablet into a now-playing display: large artwork, artist photos and bios, reviews, related albums, label info and YouTube music videos, rotating automatically.

🏷️Label explorer

Explore your collection by record label with artwork from Discogs and FanArt.tv, label merging, and every release from a chosen label one tap away.

💿Detailed album pages

High-resolution artwork, track listing, release year, label, duration, multiple artists and Pitchfork reviews where available.

🔄Self-updating

Built-in update checker with one-click updates from GitHub releases, plus share cards, dark/light themes and a clean album-first design.

Install

MusicD Remote runs as a small Docker container. Pick your setup below and the install command is built for you — copy it, paste it into a terminal, done.

1 · Where will it run?
Find it in Roon → Settings → General (or your router's device list). Left blank, the command keeps a placeholder to fill in later.
2 · Local music files
The folder on the host that contains your music library. Leave it as shown to fill in later.
Installing v1.6.35 (checking for the latest release…)
3 · Copy & run
sudo mkdir -p /opt/musicd-remote
cd /opt/musicd-remote
sudo wget https://github.com/meltface-80/MusicD-Remote/releases/download/v1.6.35/MusicD-Remote-v1.6.35.tar.gz
sudo tar -xzf MusicD-Remote-v1.6.35.tar.gz
docker build -t musicd-remote:1.6.35 .
docker run -d \
  --name musicd-remote \
  --restart unless-stopped \
  --network host \
  -v musicd-remote-data:/app/data \
  -v "/your/path/to/Music:/music:ro" \
  musicd-remote:1.6.35
If your user isn't in the docker group, prefix the docker commands with sudo.
One volume to rule them all. The musicd-remote-data volume holds your Roon pairing, play history and label cache — never rename it once created. Point every future docker run at the same name and everything carries over; a different name silently starts empty (Roon asks you to authorize again and your history is gone). Upgrading from v1.6.31 or earlier? Your data lives in the old roon-random-albums-data volume — move it once with the copy step in the Updating guide.
Docker isn't installed yet?

DietPi

sudo dietpi-software install 162

Debian / Ubuntu

curl -sSL https://get.docker.com | sh

macOS — install Docker Desktop (pick the right build for Apple Silicon or Intel), launch it, and wait for the whale icon in the menu bar to report Engine running. Then verify with docker --version.

Updating an existing install

Stop and remove the old container, download the new tarball, rebuild and re-run — keeping the same musicd-remote-data volume so pairing and history carry over. The app also shows an in-app update button when a new release is available. Full step-by-step commands are in the README Updating section.

API keys

No keys are required for basic operation — but two free keys dramatically improve label logo coverage, and a third unlocks music videos on the wall display. Each takes about two minutes. Paste them into the app under Settings (gear icon) and tap Save.

FanArt.tv API keyLabel logos

FanArt.tv provides high-quality record label logos for labels with a MusicBrainz MBID.

  1. Go to fanart.tv/get-an-api-key and register free (or log in).
  2. Follow the on-screen prompts for a personal API key — it's shown on that same page once you're signed in.
  3. Copy the key.
  4. In MusicD Remote: Settings (gear icon) → FanArt.tv key → Save.

YouTube Data API v3 keyWall display videosOptional

Lets the wall display find and play matching music videos for the current track. Free — no billing account needed.

  1. Go to console.cloud.google.com and sign in with any Google account.
  2. Create a project: project dropdown (top bar) → New project → name it anything (e.g. "MusicD Display") → Create, and make sure it's selected.
  3. Enable the API: menu → APIs & Services → Library → search "YouTube Data API v3" → open it → Enable.
  4. Create the key: APIs & Services → Credentials → + Create credentials → API key. Copy the key shown.
  5. Recommended: Edit the key → under API restrictions choose Restrict key → tick only YouTube Data API v3 → Save. That makes the key useless for anything else if it ever leaks.
  6. In MusicD Remote: Settings → YouTube API key → Save.

The free quota (10,000 units/day) comfortably covers a home display — roughly 90+ fresh tracks a day, with results cached, before it would pause until the quota resets at midnight Pacific time.

Discogs personal access tokenLabel names + logos

Discogs fills in label names that iTunes and MusicBrainz miss, and is a second source of label logos.

  1. Sign in (or register free) at discogs.com.
  2. Go to Settings → Developers → click Generate new token.
  3. Copy the token.
  4. In MusicD Remote: Settings (gear icon) → Discogs token → Save.

First run

  1. Enable the extension in Roon. Open Roon → Settings → Extensions — MusicD appears there once the container is running. Click Enable.
  2. Open the app. Browse to http://<your-server-ip>:3399 from any phone, tablet or desktop on your network (on a Mac install, http://localhost:3399 works too). Add it to your home screen for an app-like experience.
  3. Add your API keys. Tap the gear icon and paste in the keys from the section above, then let the label scan run.
  4. Optional: wall display. Point a TV or spare tablet at http://<your-server-ip>:3399/display and enjoy.
Trouble? The README troubleshooting section covers the common cases — pairing that never completes, missing zones, and label logos not appearing. Bugs and ideas are welcome on the issue tracker.
